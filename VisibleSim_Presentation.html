VisibleSim

Simulateur de Robots Modulaires Auto-Reconfigurables

Un outil de simulation pour la recherche en robotique modulaire

Qu'est-ce que VisibleSim ?

  • Simulateur open-source développé en C++17
  • Simule des systèmes de robots modulaires auto-reconfigurables
  • Supporte 6 familles de robots différentes
  • Environnement pour algorithmes distribués
  • Visualisation 3D avec OpenGL/GLUT
  • Support C++ et Meld (langage déclaratif)

Types de Robots Supportés

Architecture du Simulateur

VisibleSim/
├── simulatorCore/     # Cœur du simulateur
│   ├── base/          # Classes fondamentales
│   ├── robots/        # Implémentations spécifiques
│   ├── events/        # Système d'événements
│   ├── comm/          # Communication P2P
│   ├── gui/           # Interface graphique
│   └── motion/        # Gestion des mouvements
├── applicationsSrc/   # Code utilisateur
└── applicationsBin/   # Exécutables et configs

Classes Principales

Simulator

Point d'entrée, parse XML, initialise composants

World

Gère modules, grille spatiale, rendu graphique

BuildingBlock

Module physique avec interfaces P2P

BlockCode

Programme distribué exécuté par chaque module

Système d'Événements

  • Architecture événementielle avec ordonnanceur
  • Types d'événements :
    • Messages entre modules
    • Mouvements (rotation, translation)
    • Ajout/suppression de voisins
    • Interactions utilisateur (tap)
  • Temps simulé avec horloges locales par module
  • Support du bruit et de la variabilité

Exemple : Fichier Principal

#include "robots/blinkyBlocks/blinkyBlocksSimulator.h"
#include "monAppCode.h"

using namespace BlinkyBlocks;

int main(int argc, char **argv) {
    // Créer le simulateur
    createSimulator(argc, argv, 
                    MonAppCode::buildNewBlockCode);
    
    // Nettoyer
    deleteSimulator();
    return 0;
}

Exemple : BlockCode

class MonAppCode : public BlinkyBlocksBlockCode {
    int distance;
public:
    MonAppCode(BlinkyBlocksBlock *host) 
        : BlinkyBlocksBlockCode(host) {}
    
    void startup() override {
        // Enregistrer handler
        addMessageEventFunc2(BROADCAST_MSG,
            std::bind(&MonAppCode::handler, this, _1, _2));
        
        if (getId() == 1) {
            distance = 0;
            setColor(RED);
            sendMessageToAllNeighbors(msg, 100, 200, 0);
        }
    }
};

    // Traiter le message
}

Configuration XML - Base

<?xml version="1.0"?>
<world gridSize="20,20,20" windowSize="1024,800">
  <camera target="100,100,100" 
          directionSpherical="0,70,400" angle="45"/>
  
  <blockList color="255,255,0" ids="ORDERED">
    <block position="5,5,5"/>
    <block position="5,5,6" color="255,0,0"/>
    <blocksLine plane="XY" line="5" values="0011100"/>
  </blockList>
</world>

Configuration XML - IDs

<!-- ORDERED : IDs séquentiels (1, 2, 3, ...) -->
<blockList ids="ORDERED">
  <block position="5,5,5"/>  <!-- ID=1 -->
</blockList>

<!-- MANUAL : IDs manuels -->
<blockList ids="MANUAL">
  <block position="5,5,5" id="42"/>
</blockList>

<!-- RANDOM : IDs aléatoires -->
<blockList ids="RANDOM" seed="123" step="10">
  <block position="5,5,5"/>
</blockList>

Configuration XML - Targets

<targetList>
  <target format="grid">
    <cell position="5,5,5" color="255,0,0"/>
    <cell position="5,5,6" color="0,255,0"/>
  </target>
</targetList>

Support Meld

Langage déclaratif pour algorithmes distribués

Workflow :

  1. Écrire .meld dans applicationsBin/meld/programs/
  2. Compiler : ./compile-meld.sh programs/prog.meld
  3. Exécuter : ./meld -c config.xml -p outprogs/prog.bb -k BB

Prédicats :

!setColor(Node, R, G, B, I).
!moveTo(Node, X, Y, Z).
!at(Node, X, Y, Z).
!neighbor(NodeA, NodeB, Face).
!neighborCount(Node, Count).

Système d'Horloges

Chaque module possède une horloge interne indépendante

Modèles disponibles :

  • PerfectClock : Horloge parfaite (défaut)
  • GNoiseQClock : Modèle quadratique avec bruit gaussien
  • DNoiseQClock : Bruit basé sur données réelles
// Convertir temps simulateur → local
Time localTime = clock->getTime(simTime);

// Convertir temps local → simulateur  
Time simTime = clock->getSimulationTime(localTime);

Interactions Utilisateur

Contrôles de base :

  • Ctrl + Clic gauche : Sélectionner module
  • Ctrl + Clic droit : Menu contextuel
  • r / R : Démarrer (realtime/fastest)
  • Espace : Pause/Reprendre
  • s / S : Screenshot / Vidéo
  • q / Esc : Quitter

Menu contextuel :

  • Add/Delete block
  • Tap (événement debug)
  • Save (export XML)
  • CW/CCW Rotation (Catoms2D)

Tests de Régression

# Script de test
./utilities/blockCodeTest.sh testID ./app -c config.xml

# Dans Makefile
TESTS = ../../utilities/blockCodeTest.sh test1 $(OUT) -c c1.xml ;\
        ../../utilities/blockCodeTest.sh test2 $(OUT) -c c2.xml

# Exécuter
make test

Processus :

  1. Exécuter avec -g pour générer .confCheck.xml
  2. Renommer en .controlConf_testID.xml
  3. Tests futurs comparent avec référence
  4. Résultat : [PASS] ou [FAILED]
